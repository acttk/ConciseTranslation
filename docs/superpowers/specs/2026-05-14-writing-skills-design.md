# 中文小说写作助手 Skill 设计方案

创建日期：2026-05-14
状态：设计阶段

---

## 概述

开发两个互补的写作skill：
- **Skill A**：《仙途两难》专用写作助手，每次写新章节自动加载项目上下文
- **Skill B**：通用中文轻小说写作规范，提炼自参考小说技法，可复用于任何感情驱动题材

Skill A 调用 Skill B 的规则，加上项目专属内容。

---

## Skill A：《仙途两难》专用写作助手

### 触发条件
用户说"写第X章"、"继续写"、"续写"时触发。

### 执行流程

1. **加载上下文**
   - 读取新大纲，定位本章对应的卷/事件节点
   - 读取人物设定，确认本章出场角色的性格标签和说话方式
   - 读取写作规范（禁`——`、禁物化表达等）

2. **写出正文**
   - 每章4000-5000字
   - 对话驱动，短段落
   - 章末留钩子

3. **自检清单**
   - [ ] 无`——`
   - [ ] 无物化表达（不用"护/放/守"等支配性动词）
   - [ ] 段落均不超过5句
   - [ ] 环境描写不超过3句/处
   - [ ] 无大段心理独白
   - [ ] 每2-3章穿插云浅暗线
   - [ ] 章末有关关系变化暗示或事件悬念

### 依赖
- 新大纲文件（`仙途两难-新大纲.md`）
- 人物设定文件
- Skill B 的写作规范

## Skill B：通用中文轻小说写作规范

### 触发条件
用户写后宫/修罗场/感情驱动类小说时，或调用 Skill A 时自动加载。

### 规范条目

**1. 结构规范**
- 对话占比≥70%
- 段落1-3句为主，上限5句
- 环境描写≤3句
- 每章一个核心冲突

**2. 男主塑造**
- 温柔有底线，不舔狗
- 自嘲式幽默，话不多
- 关键时刻果断
- 能看穿女主的计算

**3. 感情推进六法**
- 食物作为情感载体
- 睡眠/夜晚场景暴露真实
- 感情靠小事累积（每章3-5个交互节点）
- 称呼渐变标记关系阶段
- 每对关系有专属互动模式
- 用潜台词和动作代替心理描写

**4. 多女主管理**
- 每角色独立物理空间
- 每角色专属互动活动
- 每角色不同的对话风格
- 冷面角色：极简对话+动作暗示情感

**5. 冲突模式**
- 微型摩擦（每章必有）
- 社交博弈（不止打架）
- 性格反差驱动

**6. 章末钩子五型**
- 关系即将变化
- 悬念信息
- 冲突预告
- 幽默反转
- 事件启动

**7. 禁止事项**
- 禁`——`连接句子
- 禁大段心理独白
- 禁物化女性表达
- 禁一段超5句
- 禁超3句环境描写
- 禁对话中空洞寒暄

### 使用方式
- 由 Skill A 调用
- 也可独立使用：`/style-check` 对照规范审查已写章节

## 两个 Skill 的关系

```
Skill B（通用规范）
    ↑ 调用
Skill A（仙途两难专用）
    ├── 加载 Skill B 规则
    ├── 加载项目大纲
    ├── 加载人物设定
    └── 执行自检清单
```

## 对话润色模块（Skill B 子功能）

写完章节后自动执行：

1. **检测书面化对话**：如"你怎么能这样对我"→口语化方向调整
2. **检测空洞寒暄**：如"你来了""嗯"→标记建议删除或加入信息量
3. **检测对话密度**：连续3句以上纯对话无动作穿插→建议插入小动作
4. **检测角色声音混淆**：不同角色的对话是否可用同一语气读出→标记
